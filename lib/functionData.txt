В системе Безопасного дорожного движения реализованы следующие функции, каждая из которых принимает определенные параметры. Ниже приведено описание параметров для каждой функции:

1. InitLedger(ctx)
Описание: Инициализация учетной записи банка и водителей.

Параметры: 

ctx - контекст транзакции, автоматически передается системой.
2. AddDrivingLicense(ctx, driverId, licenseNumber, expiryDate, category)
Описание: Добавление водительского удостоверения водителю.

Параметры:

ctx - контекст транзакции.
driverId (строка) - уникальный идентификатор водителя, для которого добавляется водительское удостоверение.
licenseNumber (строка) - номер водительского удостоверения.
expiryDate (строка) - срок действия водительского удостоверения в формате "дд.мм.гггг".
category (строка) - категория транспортного средства (A, B, C).
3. RegisterVehicle(ctx, driverId, vehicleCategory)
Описание: Регистрация транспортного средства за водителем.

Параметры:

ctx - контекст транзакции.
driverId (строка) - уникальный идентификатор водителя, который регистрирует транспортное средство.
vehicleCategory (строка) - категория транспортного средства (A, B, C).
4. RenewDrivingLicense(ctx, driverId)
Описание: Продление срока действия водительского удостоверения.

Параметры:

ctx - контекст транзакции.
driverId (строка) - уникальный идентификатор водителя, чье удостоверение необходимо продлить.
5. PayFine(ctx, driverId)
Описание: Оплата штрафа водителем.

Параметры:

ctx - контекст транзакции.
driverId (строка) - уникальный идентификатор водителя, который оплачивает штраф.
6. IssueFine(ctx, driverId)
Описание: Выписывание штрафа водителю.

Параметры:

ctx - контекст транзакции.
driverId (строка) - уникальный идентификатор водителя, которому выписывается штраф.
7. GetAllDrivers(ctx)
Описание: Получение всех зарегистрированных водителей.

Параметры:

ctx - контекст транзакции.
Общие параметры для всех функций:
ctx - предоставляется автоматически и используется для взаимодействия с блокчейн-сетью.
driverId - строка, представляющая уникальный идентификатор водителя, который может быть использован для поиска информации о водителе в системе.
licenseNumber, expiryDate, category, vehicleCategory - строки, представляющие данные о водительских удостоверениях и транспортных средствах.
Примечания:
Все даты должны быть указаны в формате "дд.мм.гггг".
В случае ошибок или некорректных данных, система выбрасывает соответствующие исключения с описанием проблемы. 
Важно, чтобы все функции проверяли наличие необходимых данных и соответствие условий перед выполнением операций.
